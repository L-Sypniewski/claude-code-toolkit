# Code Review Checklist

## Core Areas

**Correctness**: Logic, edge cases, validation, error handling, type safety
**Performance**: Algorithms, memory, DB queries, caching, async operations
**Security**: Input sanitization, SQL injection, XSS, auth/authz, no secrets, HTTPS
**Maintainability**: Naming, function length, DRY, comments, testability
**Testing**: Unit/integration tests, test quality, edge cases, mocking
**Documentation**: API docs, README, migration guides, inline comments, changelog

## Quick Wins

Remove unused imports, fix formatting, add error handling, improve naming, extract constants, validate input.
